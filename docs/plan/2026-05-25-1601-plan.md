目标：Liepin 流程在点击快捷搜索标签后，必须勾选“隐藏已查看”选项，并确保后续候选人列表读取的是该筛选生效后的结果。

改动范围：
- `src/platforms/liepin-adapter.ts`
- `src/scripts/test-liepin-adapter.ts`

实施步骤：
1. 在 `src/platforms/liepin-adapter.ts` 新增 helper，例如 `ensureLiepinHideViewedChecked(page, deadline)`。
   - 查找文本包含“隐藏已查看”的选项。
   - 判断它是否已勾选。
   - 未勾选时点击对应 checkbox / switch / label。
   - 找不到该选项时抛出带页面文本摘要的错误，避免静默跑出错误候选集。
   - 全程使用现有 `deadline` / `remainingTime(deadline)`，不新增固定长等待。

2. 调整 `openSubscribeSearch` 顺序：
   - 保持现有流程：打开 Liepin 搜索页 -> 等待 ready -> 点击 keyword 快捷搜索标签。
   - 点击标签后先等待搜索页刷新 ready。
   - 清理点击标签阶段可能缓存到的 `search-resumes` API 结果。
   - 调用 `ensureLiepinHideViewedChecked(...)`。
   - 再等待一次页面 ready，确保“隐藏已查看”触发的筛选刷新完成。
   - 返回 page 给候选人提取流程。

3. 避免 stale API fallback：
   - 因为 Liepin adapter 会观察并缓存 `search-resumes` 响应，勾选隐藏项前要清掉快捷搜索标签点击后产生的旧缓存。
   - 保留 observer，让勾选隐藏项后的新响应继续被缓存，供 `extractCandidateList` fallback 使用。

4. 增加 Liepin adapter 回归测试：
   - 验证调用顺序是：点击快捷搜索标签 -> 勾选“隐藏已查看” -> 最终 ready。
   - 验证“隐藏已查看”已经勾选时不会重复点击。
   - 验证找不到“隐藏已查看”时失败，并给出可诊断错误。
   - 保持现有 quick-search tag 失败、ready 超时、API fallback 测试不被破坏。

验证步骤：
1. `npm run typecheck`
2. `npm run test:experimental:liepin`
3. 如本地有有效 Liepin 登录态，再执行一次真实 smoke：
   `npm run smoke:liepin -- --keyword "优衣库"`
4. 人工确认 smoke 页面中快捷搜索标签点击后，“隐藏已查看”处于勾选状态，候选人列表再开始提取。
